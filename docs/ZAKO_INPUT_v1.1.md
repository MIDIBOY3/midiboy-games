# 入力システム仕様 v1.1

> Authoritative input spec (v1.1, user 2026-07-01). Hero と Zako で役割は違うが**基本操作は
> 共通**。**右親指を右スティック(高度)から離さないため、ショットはフェイスボタンからトリガーへ移動。**

## 設計思想（v1.1）

エイム技術でなく**位置取り・高度管理・戦場判断**を楽しむゲーム。右親指は常に高度操作へ専念できる
レイアウトとし、ショットはトリガー(L2/R2)へ。

## Hero / Zako の違い

- **Hero:** クラシック縦STG。ショット方向は常に**前方固定**。右スティックで照準変更はしない。"気持ちよく飛ぶ"を最優先。
- **Zako:** 基本操作は Hero と同じ。違いは**オートエイムのみ**。Zako は照準操作をしない。最も近い/脅威度の高い Hero へ自動照準。プレイヤーは立ち回り・高度・前線参加・妨害に集中。

## オートエイム（Zako）

完全追尾ではなく軽い補正：`前方約45度 → Heroが範囲内なら軽く補正 → 射撃`。"雑魚らしい射撃"を目指す。

## 操作設計思想

"照準"より"位置取り"。Hero も Zako も「どこを飛ぶか／どの高度／どの前線へ」を考えるゲーム。エイム技術でなく戦場全体を見る楽しさを優先。特に Zako は敵を倒すことより **Hero を困らせること・戦場へ貢献すること** を重視。

## 実装メモ（2026-07-01）

- 共通 InputMap を `GameState._ensure_input_actions()` で登録：`mv_left/right/up/down`（WASD/矢印＋左スティック）、`alt_up/alt_down`（右スティックY, コードで読む）、`fire/bomb/slow/boost/special/interact/decision/pause`（KB/マウス＋パッド。挙動未配線のものは "登録のみ"）。
- 高度は target＋lerp で滑らか化（ホイールは target をステップ、右スティックはアナログレート）。
- **Zako は faction-relative ビュー**：カメラを180°ロールし、Zako 移動を反転 → Hero と同じ「下から上」の縦STG感。Hero は画面上から降りてくる（＝こちらを向く）。Zako はオートエイム（マウス照準は撤去）。
- 低速移動（Shift / L1）は移動速度を約0.45倍。
- Godot のパッド定数は Xbox 命名：□=JOY_BUTTON_X, ○=JOY_BUTTON_B, △=JOY_BUTTON_Y, ×=JOY_BUTTON_A。
